### 1. СЫРЫЕ ДАННЫЕ — что получаем

**Records (записи)** — основной источник, ~300/день для салона BrowUp
```
id, date, datetime, staff_id, client_id, services[], cost, attendance,
online, record_from, create_date, seance_length, deleted, paid_full,
payment_status, finance_transactions[], goods_transactions[],
consumables[], visit_id, comment, confirmed, bookform_id
```

**Clients (клиенты)** — 27,406 в базе
```
id, name, phone, email, birth_date, sex, discount, visits_count,
last_visit_date, spent, paid, balance, importance_id, categories[tags],
custom_fields, card (loyalty)
```

**Staff (мастера)** — 134 сотрудника
```
id, name, specialization, position{id,title}, rating, votes_count,
avatar, fired, bookable, schedule_till, weight, status, hidden
```

**Services (услуги)** — 353 услуги
```
id, title, category_id, price_min, price_max, discount, active,
staff[] (кто оказывает), duration, is_multi, capacity
```

**Analytics Overall** — готовые агрегаты за период
```
income_total_stats     {current_sum, previous_sum, change_percent}
income_services_stats  {current_sum, previous_sum, change_percent}
income_goods_stats     {current_sum, previous_sum, change_percent}
income_average_stats   {средний чек}
fullness_stats         {current_percent, previous_percent}
record_stats           {completed, pending, canceled, total + vs previous}
client_stats           {total, new, return, active, lost + проценты}
```

**Charts** — массивы по дням (timestamps)
```
income_daily      → [{timestamp, amount}] × 3 серии (total/services/goods)
fullness_daily    → [{timestamp, percent}] × 2 серии (загрузка/no-show)
records_daily     → [{timestamp, count}]
record_source     → [{label, count}]  (ресепшн, онлайн, app...)
record_status     → [{label, count}]  (пришёл, ожидает, подтверждён)
```

**Finance**
```
accounts[]             — кассы (5 штук: нал, безнал, ЗП, Каспий...)
transactions[]         — каждая транзакция: amount, date, account, client, master
z_report               — дневной отчёт: клиенты, ср.чек, оплаты по кассам, детализация
```

**Loyalty**
```
programs[]             — 8 программ (кэшбэк 2%, VIP, реферал...)
card_types[]           — 2 типа карт лояльности
membership_types[]     — 23 типа абонементов (эндосфера, dermadrop...)
certificate_types[]    — 25 типов сертификатов (10K-50K₸)
```

**Schedule**
```
staff_schedule[]       — 69 слотов расписания за период
timetable_dates[]      — 49 дат доступных для записи
resources[]            — 4 ресурса (кресла, столы...)
```

**Other**
```
storages[]             — 6 складов + транзакции расходников
goods[]                — товары с ценами
users[]                — 79 пользователей системы
roles[]                — 8 ролей (Owner, Manager, Receptionist...)
permissions{}          — 11 групп прав
license                — тариф Unlimited, до 2027, 945K₸/год
notification_settings  — 11 типов уведомлений
custom_fields          — кастомные поля для записей и клиентов
```

### 2. ЧТО МОЖНО ВЫЧИСЛИТЬ (derived metrics)

**По мастерам** (из records):
| Метрика | Как считать |
|---------|------------|
| Выручка мастера | `sum(services.cost) WHERE staff_id = X AND attendance = 2` |
| Средний чек мастера | выручка / количество визитов |
| Загрузка мастера | часы визитов / часы по расписанию × 100 |
| % отмен мастера | cancelled / total × 100 |
| Кол-во клиентов | `COUNT(DISTINCT client_id)` |
| Кол-во новых клиентов | клиенты с `is_new=true` |
| Возвратность | клиенты с >1 визита / всего клиентов |
| Средний расход расходников | consumables per visit |
| Выручка/час | revenue / worked_hours |

**По клиентам** (из records + clients):
| Метрика | Как считать |
|---------|------------|
| LTV клиента | `sum(cost)` за всю историю |
| Частота визитов | визитов / месяцев с первого визита |
| Средний чек | total_spent / visits_count |
| Дней с последнего визита | `today - last_visit_date` |
| Риск оттока | если дней > 2× средний интервал |
| Любимый мастер | самый частый staff_id |
| Любимая услуга | самая частая service |
| Сегмент | VIP (топ 10% LTV) / Regular / New / At-risk / Lost |

**По услугам** (из records):
| Метрика | Как считать |
|---------|------------|
| Популярность | count записей за период |
| Выручка по услуге | sum(cost) |
| Средняя скидка | avg(discount) |
| Конверсия | completed / total |
| Кросс-продажи | какие услуги берут вместе |
| Себестоимость | из consumables per service |
| Маржинальность | (revenue - consumables_cost) / revenue |

**По бизнесу**:
| Метрика | Как считать |
|---------|------------|
| Потерянная выручка от отмен | cancelled × avg_bill |
| Упущенная выручка от простоя | (100% - fullness%) × capacity × avg_revenue_per_hour |
| Потенциал от возврата потерянных | lost_clients × avg_ltv × 10% return_rate |
| Эффективность онлайн-записи | online_records / total × 100 |
| Сезонность | выручка по месяцам/неделям |
| Пиковые часы | records grouped by hour |
| Лучший день недели | records grouped by weekday |

### 4. ВАРИАНТЫ MVP

#### MVP A: "Утренний отчёт" (Telegram бот)
**Что**: каждое утро владелец получает сводку в Telegram
**Сложность**: низкая (2-3 недели)
**Монетизация**: $10-15/мес

```
Что отправляем:
├── Вчера: выручка, визиты, ср.чек, сравнение с прошлой неделей
├── Топ-3 мастера по выручке
├── Проблемы: кто из мастеров просел, сколько отмен
├── Сегодня: загрузка, пустые слоты, количество записей
└── Неделя: тренд выручки, новые vs потерянные клиенты
```

**Плюсы**: минимальный порог входа, не надо заходить в дашборд, мгновенная ценность
**Минусы**: пассивный продукт, сложно масштабировать цену, через месяц могут перестать читать

---

#### MVP B: "Потери выручки" (одностраничный отчёт)
**Что**: автогенерируемый отчёт "где вы теряете деньги" с конкретными суммами
**Сложность**: средняя (3-4 недели)
**Монетизация**: $15-25/мес

```
Отчёт содержит:
├── Потери от отмен: 1,324 × 33,755₸ = ~44.7M₸ потенциал
├── Потери от простоя: 46.5% × мощность = X₸/мес
├── Потери от оттока: 554 клиента × avg LTV = X₸
├── Мастера-проблемы: кто тянет вниз (отмены, низкий чек, мало загрузки)
├── Мастера-звёзды: кого нагружать больше
├── Неэффективные часы: когда пусто, когда перегруз
├── Товары: продажи -15%, что падает конкретно
└── Рекомендации: 3-5 конкретных действий с ожидаемым эффектом
```

**Плюсы**: сильный wow-эффект ("мы теряем 44M₸?!"), actionable
**Минусы**: нужен красивый frontend, разовый wow может угаснуть

---

#### MVP C: "Отчёт по мастерам" (дашборд)
**Что**: таблица/рейтинг мастеров с ключевыми метриками
**Сложность**: средняя (3-4 недели)
**Монетизация**: $15-25/мес

```
По каждому мастеру:
├── Выручка (абсолют + тренд)
├── Средний чек
├── Количество визитов
├── Загрузка (% от расписания)
├── % отмен
├── Кол-во новых клиентов
├── Возвратность клиентов
├── Выручка/час (эффективность)
├── Средний расход расходников
└── Сравнение с прошлым периодом + с другими мастерами
```

**Плюсы**: владельцы реально хотят это (кто работает, кто нет), основа для решений по ЗП/увольнению
**Минусы**: Altegio уже имеет базовую аналитику по мастерам (но слабую)

---

#### MVP D: "AI-консультант" (чат-бот)
**Что**: задай вопрос про свой бизнес — получи ответ с цифрами
**Сложность**: средняя-высокая (4-6 недель)
**Монетизация**: $20-35/мес

```
Примеры вопросов:
├── "Как дела в этом месяце?"
├── "Кто из мастеров просел за последнюю неделю?"
├── "Сколько мы теряем на отменах?"
├── "Какие клиенты давно не приходили?"
├── "Какие услуги приносят больше всего?"
├── "Стоит ли нанять ещё одного nail мастера?"
└── "Сравни март и апрель"
```

**Плюсы**: wow-эффект, высокая perceived value, sticky product
**Минусы**: сложнее реализовать, качество ответов критично, дороже в эксплуатации (Claude API costs)

---

#### MVP E: "Клиенты в зоне риска" (автоматизация возвратов)
**Что**: система находит клиентов которые скоро уйдут и предлагает действия
**Сложность**: средняя (3-4 недели)
**Монетизация**: $15-25/мес

```
Сегментация:
├── VIP (топ 10% LTV)          → персональное внимание
├── Постоянные                  → поддерживаем
├── Новые (1-2 визита)          → критично вернуть на 3-й визит
├── В зоне риска (давно не был) → триггер: "не были X дней"
└── Потерянные (90+ дней)       → кампания возврата

Действия:
├── Список клиентов для обзвона (с контекстом: что делали, когда, у кого)
├── Готовые SMS/WhatsApp шаблоны
└── Метрика: сколько вернули → сколько заработали
```

**Плюсы**: прямая связь с деньгами (вернули клиента = конкретная выручка)
**Минусы**: нужна интеграция с SMS/WhatsApp для полного эффекта, или просто список для ручного обзвона

### 5. РЕКОМЕНДАЦИЯ

**Комбо: B + C + утренний Telegram = "всё в одном"**

Почему:
- **Telegram-бот** — daily hook, напоминание о продукте каждое утро
- **Отчёт по мастерам** — причина номер 1 почему владельцы смотрят аналитику
- **Потери выручки** — wow при онбординге и еженедельно

Это покрывается **одними и теми же данными** (records + analytics + staff), одним sync процессом, одним бэкендом.

Но прежде всего — **поговори с 5 владельцами**. Покажи им сгенерированный отчёт по их данным и спроси: "за это вы бы платили 5000₸/мес?" Реакция скажет больше чем любой план.
