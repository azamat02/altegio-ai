# Altegio AI — Roadmap

> **Золотое правило:** каждая фаза гейтится деньгами клиентов. Не строим спеки Phase N+1 до того как есть платящие за Phase N. Это черновой roadmap — формальные спеки и планы пишутся per-phase по факту готовности рынка.

---

## Phase 1 — Telegram Morning Report ✅ (локально готово)

**Статус:** полностью реализовано локально, 41 коммит, 44/44 unit + 6/7 integration тестов. Остался deploy на VPS (`altegio.tolemflow.kz` → `178.128.202.65`), домен указывает.

**Спек:** [`docs/superpowers/specs/2026-04-20-altegio-ai-phase-1-design.md`](docs/superpowers/specs/2026-04-20-altegio-ai-phase-1-design.md)
**План:** [`docs/superpowers/plans/2026-04-20-altegio-ai-phase-1.md`](docs/superpowers/plans/2026-04-20-altegio-ai-phase-1.md)

**Что это:** одно Telegram-сообщение в 09:00 по TZ салона с вчерашней выручкой, топ-3 мастерами, алертами (рост отмен, мастер в просадке), прогнозом на сегодня и AI-инсайтом от Claude (с guardrails против галлюцинаций).

**Монетизация:** $10-15/мес.

**Гейт на Phase 2:** 3-5 платящих салонов на Phase 1.

## Phase 2 — Losses Report (когда есть платящие за Phase 1)

**Концепция:** "Вы теряете 44.7M ₸ в год — вот на чём конкретно". Wow-отчёт для онбординга и еженедельной досылки.

**Формат:** веб-страница (TG Web App) + PDF + ссылка в Telegram. Триггерится либо по кнопке в боте, либо автоматом раз в неделю.

**Содержимое:**
```
1. Потери от отмен:      cancelled × avg_bill (годовой проекция)
2. Потери от простоя:    (100% - occupancy%) × capacity × revenue_per_hour
3. Потери от оттока:     lost_clients × avg_LTV × реалистичный % возврата
4. Мастера-проблемы:     кто тянет вниз (отмены, низкий чек, мало загрузки)
5. Мастера-звёзды:       кого догружать больше
6. Неэффективные часы:   когда пусто vs когда перегруз
7. Товары:               продажи товаров падают, что именно и почему
8. Рекомендации:         3-5 конкретных actions с ожидаемым эффектом в ₸
```

**Стек поверх Phase 1:** Next.js TG Web App для веб-части + puppeteer для PDF + новый `LossesService` поверх существующей `MetricsService`. Данные уже есть — нужен только новый агрегатор + рендер.

**Сроки:** 2-3 недели соло.

**Монетизация:** апгрейд-подписка, $25-35/мес.

**Гейт на Phase 3:** 15-20 платящих + запросы "а можно посмотреть больше данных / глубже копнуть".

## Phase 3 — Masters Dashboard (Telegram Mini App)

**Концепция:** полноценный дашборд внутри Telegram, без отдельного логина. Продолжение пути "всё в Telegram" из Phase 1/2.

**Что внутри:**
- Таблица мастеров с 10+ метриками (выручка, ср.чек, загрузка, % отмен, новые клиенты, возвратность, выручка/час, расход расходников)
- Sparkline тренда по каждому мастеру за 30 дней
- Drill-down при клике: клиенты мастера, его услуги, его расписание
- Сравнение периодов (этот месяц vs прошлый, эта неделя vs прошлая)
- Фильтры: по услуге, по дню недели, по диапазону дат

**Почему TMA а не веб:**
- Auth через Telegram (initData) — нет регистрации/паролей
- Нативно на мобильном, владельцы смотрят с телефона
- Один тап из утреннего отчёта → открывается дашборд

**Стек:** Next.js 15 + Tremor/Recharts + Telegram WebApp SDK + API-слой поверх существующих сервисов.

**Сроки:** 3-4 недели.

**Монетизация:** $35-50/мес.

**Гейт на Phase 4:** $2-3K MRR или явный запрос клиентов на AI-консультанта / retention.

## Phase 4 — AI Consultant (MVP D из первоначального обсуждения)

**Концепция:** chat с Claude внутри Telegram. Владелец задаёт вопросы — получает ответы на цифрах его салона.

**Примеры вопросов:**
- "Как дела в этом месяце?"
- "Стоит ли мне нанять ещё одного nail-мастера?"
- "Кто из клиентов давно не приходил и сколько я потерял?"
- "Сравни март и апрель"
- "Почему в среду всегда просадка?"

**Техническая сложность:** RAG над данными салона, careful prompt engineering, стоимость per-call. Сроки: 4-6 недель. Цена: $50-80/мес. Sticky продукт с высоким perceived value.

**Риск:** качество ответов критично — плохой ответ = отписка.

## Phase 5 — Retention Automation (MVP E)

**Концепция:** находим клиентов в зоне оттока и даём готовые SMS/WhatsApp-шаблоны для обзвона.

**Сегментация:**
- VIP (топ 10% LTV) — персональное внимание
- Постоянные — поддерживаем
- Новые (1-2 визита) — критично вернуть на 3-й
- В зоне риска (давно не был) — триггер "не были X дней"
- Потерянные (90+ дней) — win-back кампания

**Выход:** список клиентов для обзвона с контекстом (что делали, у кого, когда) + готовые SMS/WhatsApp-шаблоны. Метрика "вернули X → заработали Y ₸" — прямая связь с деньгами.

**Сроки:** 3-4 недели. Нужна интеграция с SMS/WhatsApp Business API или хотя бы список для ручного обзвона.

## Longer-term идеи (не коммититься пока)

- **Multi-location chains** — владелец сети из 5 салонов видит сводку по всем + drill-down на каждый
- **Anonymous benchmarks** — "ваш средний чек ниже 73% похожих салонов в городе" (агрегация по платформе)
- **Прогнозы** — "ожидаемая выручка на следующей неделе", "какой день самый слабый"
- **Altegio Marketplace** — выходить как официальное приложение (OAuth вместо логина/пароля), self-service онбординг
- **Biliing** — Stripe/Kaspi.kz, автоматическая подписка (до этого — вручную инвойсы)

## Что может убить проект

1. **Altegio меняет API** — V1 закрывает, V2 ломает. Митигация: быть близко к Altegio, следить за changelog, возможно стать partner-разработчиком.
2. **Сами делают аналитику лучше** — Altegio Marketplace в analytics-категории почти пуст (наша entry point), но могут раскачать свою. Риск средний.
3. **Низкий спрос** — "владельцы не хотят платить за аналитику, они хотят больше клиентов". Customer development покажет. Если не платят — пивотимся в Phase 5 (retention = прямая связь с деньгами) или пересматриваем продукт.
4. **Rate limits Altegio** (200/мин) — при росте до 50+ салонов можем упереться. Митигация: оптимизировать sync (меньше delta window, webhooks когда появятся).

## Принципы построения продукта

**Не строим то, что не оплачивают.** Формальные спеки Phase 2+ пишем только когда есть 3-5 платящих за предыдущую фазу.

**Продажа > код.** Соло-основатель должен тратить 50%+ времени на звонки, не на рефакторинг.

**Всё в Telegram, пока возможно.** Нативность + отсутствие auth = низкий порог входа. Веб/PDF только когда Telegram не тянет формат (Phase 2 losses-отчёт с графиками).

**AI — поверх метрик, не вместо.** Claude комментирует цифры, не считает их. Формулы детерминированы, AI — voice layer.

**Multi-tenant с первого дня.** Избегаем rewrite-боли. Self-service online-регистрация — позже.
